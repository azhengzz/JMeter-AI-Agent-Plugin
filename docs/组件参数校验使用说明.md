# 组件参数校验功能使用说明

## 概述

组件参数校验功能通过 YAML 配置文件定义 JMeter 组件的参数规则，在创建组件时自动校验参数是否正确传递。

## 工作原理

1. **YAML Schema 定义**：在 `references/` 目录下创建 `.schema.yaml` 文件定义组件参数
2. **自动加载**：`ComponentSchemaLoader` 启动时自动扫描并加载所有 schema 文件
3. **参数校验**：创建组件时，`ComponentValidator` 根据 schema 校验参数
4. **向后兼容**：未配置 schema 的组件跳过校验，不影响现有功能

## Schema 文件格式

### 文件位置
```
src/main/jmeter-agent/skills/jmeter/references/{组件类型}/{组件名称}.schema.yaml
```

### Schema 结构

```yaml
component:
  type: durationassertion           # 组件类型（标准化）
  name: Duration Assertion          # 组件显示名称
  description: 验证响应时间          # 组件描述

properties:
  - name: DurationAssertion.duration   # JMeter 属性名
    type: Integer                        # 参数类型
    required: true                       # 是否必传
    min: 0                               # 最小值（数值类型）
    max: 2147483647                      # 最大值（数值类型）
    description: 最大允许持续时间        # 参数描述

  - name: HTTPSampler.method
    type: String
    required: false
    enum: ["GET", "POST", "PUT", "DELETE"]   # 允许的枚举值
    default: "GET"                        # 默认值

  - name: HTTPSampler.domain
    type: String
    required: false
    pattern: "^[a-zA-Z0-9.-]+$"           # 正则表达式校验
```

## 支持的参数类型

| 类型 | 说明 | 示例值 |
|------|------|--------|
| `String` | 字符串 | `"hello"`, `"https"` |
| `Integer` | 整数 | `100`, `8080` |
| `Boolean` | 布尔值 | `true`, `false` |
| `Number` | 数值（含小数） | `3.14`, `100.5` |
| `Object` | 对象（Map结构） | `{"key": "value"}` |
| `Array` | 数组 | `["a", "b", "c"]` |

## 支持的校验规则

| 规则 | 类型 | 说明 |
|------|------|------|
| `required` | 全部 | 是否必传参数 |
| `enum` | String | 允许的枚举值列表 |
| `min` | Integer | 最小值 |
| `max` | Integer | 最大值 |
| `pattern` | String | 正则表达式校验 |
| `default` | 全部 | 默认值 |

## 已创建的 Schema 文件

| 组件 | Schema 文件 | 必传参数 |
|------|-------------|----------|
| Duration Assertion | `assertions/DurationAssertion.schema.yaml` | `DurationAssertion.duration` |
| Response Assertion | `assertions/ResponseAssertion.schema.yaml` | `Assertion.test_field`, `Assertion.test_type`, `Assertion.test_string` |
| JSON Path Assertion | `assertions/JSONPathAssertion.schema.yaml` | `JSON_PATH_ASSERTION.jsonPath` |
| HTTP Request | `samplers/HTTP Request.schema.yaml` | 无（domain/path 由 HTTP Defaults 提供） |
| Constant Timer | `timers/ConstantTimer.schema.yaml` | `ConstantTimer.delay` |

## 校验错误示例

### 缺少必传参数
```
**Component Parameter Validation Failed**

Component Type: durationassertion

Errors:
  - Missing required parameter 'DurationAssertion.duration'. Maximum allowed duration in milliseconds

**Solution**: Please provide all required parameters with correct values.
Refer to component documentation for parameter details.
```

### 参数值类型错误
```
**Component Parameter Validation Failed**

Component Type: httpsampler

Errors:
  - Parameter 'HTTPSampler.port' has invalid type. Expected INTEGER but got String

**Solution**: Please provide all required parameters with correct values.
Refer to component documentation for parameter details.
```

### 枚举值不在列表内
```
**Component Parameter Validation Failed**

Component Type: responseassertion

Errors:
  - Parameter 'Assertion.test_field' has invalid value 'Invalid'. Allowed values: [Response Code, Response Message, Response Data, Response Headers, Request Data, Request Headers]

**Solution**: Please provide all required parameters with correct values.
Refer to component documentation for parameter details.
```

## 扩展指南

### 为新组件添加校验

1. 在对应目录创建 `.schema.yaml` 文件
2. 定义组件信息和参数规则
3. 重新构建项目（或重启 JMeter）
4. 创建组件时自动应用校验

### 示例：添加 LoopController 校验

创建 `src/main/jmeter-agent/skills/jmeter/references/controllers/LoopController.schema.yaml`：

```yaml
component:
  type: loopcontroller
  name: Loop Controller
  description: Loops a specified number of times

properties:
  - name: LoopController.loops
    type: Integer
    required: false
    min: -1
    default: 1
    description: Number of loops (-1 = forever)

  - name: LoopController.continue_forever
    type: Boolean
    required: false
    default: false
    description: Loop forever
```

## 核心类说明

| 类 | 路径 | 说明 |
|------|------|------|
| `ComponentSchema` | `validation/ComponentSchema.java` | Schema 数据模型 |
| `ComponentSchemaLoader` | `validation/ComponentSchemaLoader.java` | Schema 加载器 |
| `ComponentValidator` | `validation/ComponentValidator.java` | 参数校验器 |
| `CreateJMeterElementTool` | `tools/jmeter/CreateJMeterElementTool.java` | 集成校验的工具类 |

## 注意事项

1. **向后兼容**：未配置 schema 的组件不会进行校验，现有功能不受影响
2. **类型映射**：YAML 中的字符串会自动转换为对应的 Java 类型
3. **可选参数**：`required: false` 的参数可以不传递
4. **空值处理**：`null` 值在可选参数校验中通过
5. **枚举大小写**：枚举值校验默认区分大小写
