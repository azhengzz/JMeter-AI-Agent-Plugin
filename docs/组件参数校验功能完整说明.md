## 1. 概述

组件参数校验功能通过 YAML 配置文件定义 JMeter 组件的参数规则，在创建组件时自动校验参数是否正确传递，并支持默认值自动应用。

## 2. 需求背景

在创建 JMeter 组件时，需要添加参数校验功能：
1. 通过 YAML 文件配置组件参数定义
2. 校验必传参数是否完整
3. 校验可选参数值是否正确（类型、枚举值、范围等）
4. 自动应用 schema 中定义的默认值
5. 只对 YAML 中配置的组件进行强制校验
6. 兼容现有创建组件逻辑（未配置 schema 的组件跳过校验）

## 3. 实现方案

### 3.1 核心架构

```
YAML Schema → ComponentSchemaLoader → ComponentSchema → ComponentValidator → ValidationResult
```

### 3.2 工作流程

```
用户传入参数
    ↓
加载组件 Schema
    ↓
参数校验（必传、类型、枚举、范围）
    ↓
应用默认值
    ↓
创建组件
```

## 4. Schema 文件格式

### 4.1 文件位置

```
src/main/jmeter-agent/skills/jmeter/references/{组件类型}/{组件名称}.schema.yaml
```

### 4.2 Schema 结构

```yaml
component:
  type: httpsampler           # 组件类型（标准化）
  name: HTTP Request          # 组件显示名称
  description: 发送 HTTP/HTTPS 请求

properties:
  - name: HTTPSampler.protocol
    type: String
    required: false
    enum: ["http", "https", "file"]
    default: "http"
    description: 协议类型

  - name: HTTPSampler.domain
    type: String
    required: false
    description: 域名或 IP 地址

  - name: HTTPSampler.port
    type: Integer
    required: false
    min: 1
    max: 65535
    default: 80
    description: 端口号

  - name: HTTPSampler.method
    type: String
    required: false
    enum: ["GET", "POST", "PUT", "DELETE"]
    default: "GET"
    description: HTTP 方法

  - name: HTTPSampler.postBodyRaw
    type: Boolean
    required: false
    default: false
    description: 发送原始请求体

  - name: Assertion.test_strings
    type: Array
    required: false
    description: 多个测试字符串（集合）
```

## 5. 支持的功能

### 5.1 参数类型

| 类型 | Java 检查 | 示例值 |
|------|-----------|--------|
| `String` | `instanceof String` | `"hello"`, `"https"` |
| `Integer` | `instanceof Integer/Long` | `100`, `8080` |
| `Boolean` | `instanceof Boolean` | `true`, `false` |
| `Number` | `instanceof Number` | `3.14`, `100.5` |
| `Object` | `instanceof Map` | `{"key": "value"}` |
| `Array` | `instanceof Iterable or array` | `["a", "b", "c"]` |

### 5.2 校验规则

| 规则 | 类型 | 说明 |
|------|------|------|
| `required` | 全部 | 是否必传参数 |
| `enum` | String | 允许的枚举值列表 |
| `min` | Integer | 最小值 |
| `max` | Integer | 最大值 |
| `pattern` | String | 正则表达式校验 |
| `default` | 全部 | 默认值（未指定时自动应用） |

### 5.3 已创建的 Schema 文件

| 组件 | Schema 文件 | 必传参数 |
|------|-------------|----------|
| Duration Assertion | `assertions/DurationAssertion.schema.yaml` | `DurationAssertion.duration` |
| Response Assertion | `assertions/ResponseAssertion.schema.yaml` | `Assertion.test_field`, `Assertion.test_type` |
| JSON Path Assertion | `assertions/JSONPathAssertion.schema.yaml` | `JSON_PATH_ASSERTION.jsonPath` |
| HTTP Request | `samplers/HTTP Request.schema.yaml` | 无（domain/path 由 HTTP Defaults 提供） |
| Constant Timer | `timers/ConstantTimer.schema.yaml` | `ConstantTimer.delay` |

## 6. 核心类设计

### 6.1 ComponentSchema.java

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/validation/ComponentSchema.java`

```java
public class ComponentSchema {
    private String componentType;
    private String componentName;
    private String description;
    private final List<PropertyDefinition> properties = new ArrayList<>();

    public static class PropertyDefinition {
        private String name;
        private PropertyType type;        // STRING, INTEGER, BOOLEAN, NUMBER, OBJECT, ARRAY
        private boolean required;
        private Object defaultValue;
        private List<String> enumValues;
        private Integer minValue;
        private Integer maxValue;
        private String pattern;
        private String description;
    }

    public enum PropertyType {
        STRING, INTEGER, BOOLEAN, NUMBER, OBJECT, ARRAY
    }
}
```

### 6.2 ComponentSchemaLoader.java

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/validation/ComponentSchemaLoader.java`

**职责**：
- 从 `references/` 目录加载 `.schema.yaml` 文件
- 使用 **SnakeYAML** 解析 YAML
- 缓存已加载的 schema（按标准化组件类型索引）

**关键方法**：
- `loadSchema(String elementType)`: 返回 ComponentSchema 或 null（未配置时）
- `loadAllSchemas()`: 启动时扫描所有 schema 文件

### 6.3 ComponentValidator.java

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/validation/ComponentValidator.java`

**校验顺序**：
1. **未知参数检查** - 参数是否在 schema 中定义（显示可用参数列表）
2. **必传参数检查** - required 参数是否存在
3. **类型检查** - 参数值类型是否正确
4. **值检查** - 枚举、范围、正则等约束

**向后兼容**：如果组件没有 schema，返回 `ValidationResult.valid()`

**新增方法**：
- `getSchemaLoader()`: 返回 schema loader 实例，用于访问 schema 定义

### 6.4 CreateJMeterElementTool.java 集成

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/tools/jmeter/CreateJMeterElementTool.java`

**集成点**：在 `executeInternal()` 方法中，基本校验之后、组件创建之前

```java
// Schema validation (if schema exists)
if (componentValidator != null) {
    ValidationResult validation = componentValidator.validate(elementType, properties);
    if (!validation.isValid()) {
        String errorMsg = buildValidationErrorMessage(elementType, validation);
        return ToolResult.error(errorMsg);
    }

    // Apply default values from schema
    properties = applyDefaultValues(elementType, properties);
}
```

**新增方法**：
- `applyDefaultValues(String elementType, Map<String, Object> properties)`: 应用 schema 中定义的默认值
- `mapPropertyName(String propName)`: 处理 JMeter 历史拼写错误的属性名映射

## 7. 数组属性支持

### 7.1 背景

用户需要传递数组类型的属性，如 `Assertion.test_strings=["200", "301"]`，这在 JMeter 中对应 `CollectionProperty`。

### 7.2 实现方式

在 `CreateJMeterElementTool.setElementProperties()` 中添加数组检测和处理逻辑：

```java
// 使用反射创建 CollectionProperty 和 StringProperty
Class<?> collectionPropClass = Class.forName("org.apache.jmeter.testelement.property.CollectionProperty");
Class<?> stringPropClass = Class.forName("org.apache.jmeter.testelement.property.StringProperty");

// 创建 CollectionProperty（需要两个参数）
Object collectionProp = collectionPropClass
    .getConstructor(String.class, java.util.Collection.class)
    .newInstance(propName, new java.util.ArrayList<>());

// 使用 addProperty 方法添加元素
java.lang.reflect.Method addPropertyMethod = collectionPropClass.getMethod("addProperty", jmeterPropClass);
```

### 7.3 生成的 XML 结构

```xml
<collectionProp name="Assertion.test_strings">
  <stringProp name="1712478901123">200</stringProp>
  <stringProp name="1712478901124">301</stringProp>
</collectionProp>
```

## 8. 默认值自动应用

### 8.1 功能说明

如果参数未指定但 schema 中定义了默认值，系统会自动应用默认值，提升用户体验。

### 8.2 两种 Default 值的区别

理解 schema 中的 `default` 与 JMeter 内部默认值的差异非常重要：

| 类型 | 定义位置 | 作用时机 | 处理方式 |
|------|---------|---------|----------|----------|
| **Schema default** | YAML schema 文件 | AI 调用时参数补充 | 如果 AI 未传参数，系统自动补充 |
| **JMeter 默认值** | JMeter 源码内部 | 组件初始化时 | JMeter 组件自己设置 |

### 8.3 实际处理流程

```
AI 调用（未传 protocol）
    ↓
检查 Schema default
    ↓
如果 Schema 有 default → 补充参数
    ↓
如果 Schema 无 default → 不补充
    ↓
创建组件
    ↓
JMeter 内部默认值 → 覆盖或使用
```

### 8.4 示例对比

#### HTTPSampler.protocol

- **Schema default**: 已移除（不定义 `default`）
- **JMeter 默认值**: JMeter 自动设置为 "http"
- **结果**: AI 不传 protocol 时，依赖 JMeter 的默认值

#### HTTPSampler.method

- **Schema default**: `"GET"`
- **JMeter 默认值**: 可能没有或不确定
- **结果**: AI 不传 method 时，系统补充 "GET"

### 8.5 最佳实践

1. **优先使用 JMeter 默认值**：如果 JMeter 组件本身有合理的默认值（如 protocol="http"），不要在 schema 中重复定义
2. **补充 JMeter 缺少的默认值**：对于 JMeter 没有默认值但业务上应该有默认值的参数（如 method="GET"），在 schema 中定义 default
3. **避免冲突**：确保 Schema default 与 JMeter 默认值一致，避免混淆

### 8.6 使用示例

**Schema 定义**（HTTPSampler.protocol 无 default，依赖 JMeter）：
```yaml
- name: HTTPSampler.protocol
  type: String
  required: false
  enum: ["http", "https", "file"]
  # 依赖 JMeter 默认值，不在 schema 中定义 default

- name: HTTPSampler.method
  type: String
  required: false
  enum: ["GET", "POST", "PUT", "DELETE"]
  default: "GET"
```

**用户调用**（未传 protocol 和 method）：
```json
{
  "elementType": "httpsampler",
  "elementName": "GET请求",
  "properties": {
    "HTTPSampler.domain": "api.example.com",
    "HTTPSampler.path": "/api/users"
  }
}
```

**处理结果**：
```json
{
  "HTTPSampler.domain": "api.example.com",
  "HTTPSampler.path": "/api/users",
  "HTTPSampler.method": "GET"        // Schema default 补充
}
// protocol 未补充，依赖 JMeter 默认值
```

**日志输出**：
```
INFO Applied default value for HTTPSampler.method: GET
// protocol 没有日志，因为 schema 中没有定义 default
```

## 10. 问题与解决

### 10.1 PropertyDefinition.getName() 返回 null

**问题**：YAML 解析时某些属性定义的 name 字段为空

**解决**：
1. 在 `parsePropertyField` 中跳过空 name
2. 在 `getProperty` 中添加 null 检查
3. 在 `getRequiredProperties` 中过滤无效属性

### 10.2 properties 空值处理

**问题**：`Map.of()` 创建的不可变 Map 导致 NPE

**解决**：改用 `new HashMap<>()` 创建可变 Map

### 10.3 CollectionProperty 构造函数错误

**问题**：`NoSuchMethodException: CollectionProperty.<init>(java.lang.String)`

**解决**：使用两参数构造函数 `CollectionProperty(String name, Collection<?> value)`

### 10.4 StringProp 类名错误

**问题**：`ClassNotFoundException: StringProp`

**解决**：正确类名是 `StringProperty`

### 10.5 addItem 方法签名错误

**问题**：`NoSuchMethodException: addItem(JMeterProperty)`

**解决**：使用 `addProperty(JMeterProperty)` 方法

### 10.6 属性名拼写错误

**问题**：JMeter 历史遗留拼写 `Asserion.test_strings`（少个 't'）

**解决**：在 `mapPropertyName()` 中添加从 `Assertion.test_strings` 到 `Asserion.test_strings` 的映射


## 12. 扩展指南

### 12.1 为新组件添加校验

1. 在对应目录创建 `.schema.yaml` 文件
2. 定义组件信息和参数规则
3. 重新构建项目（或重启 JMeter）
4. 创建组件时自动应用校验

### 12.2 示例：添加 LoopController 校验

创建 `src/main/jmeter-agent/skills/jmeter/references/controllers/LoopController.schema.yaml`：

```yaml
component:
  type: loopcontroller
  name: Loop Controller
  description: Loops a specified number of times

properties:
  - name: LoopController.loops
    type: Integer
    required: false
    min: -1
    default: 1
    description: Number of loops (-1 = forever)

  - name: LoopController.continue_forever
    type: Boolean
    required: false
    default: false
    description: Loop forever
```

## 13. 注意事项

1. **向后兼容**：未配置 schema 的组件不会进行校验，现有功能不受影响
2. **类型映射**：YAML 中的字符串会自动转换为对应的 Java 类型
3. **可选参数**：`required: false` 的参数可以不传递
4. **空值处理**：`null` 值在可选参数校验中通过
5. **枚举大小写**：枚举值校验默认区分大小写
6. **默认值应用**：如果参数未指定但 schema 中定义了默认值，会自动应用默认值
7. **属性名映射**：系统自动处理 JMeter 历史拼写错误（如 `Assertion.test_strings` → `Asserion.test_strings`）
