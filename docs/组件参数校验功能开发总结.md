# 组件参数校验功能开发总结

## 1. 需求背景

在创建 JMeter 组件时，需要添加参数校验功能：
1. 通过 YAML 文件配置组件参数定义
2. 校验必传参数是否完整
3. 校验可选参数值是否正确（类型、枚举值、范围等）
4. 只对 YAML 中配置的组件进行强制校验
5. 兼容现有创建组件逻辑（未配置 schema 的组件跳过校验）

## 2. 实现方案

### 2.1 核心架构

```
YAML Schema → ComponentSchemaLoader → ComponentSchema → ComponentValidator → ValidationResult
```

### 2.2 YAML Schema 文件格式

**文件位置**：`src/main/jmeter-agent/skills/jmeter/references/{组件类型}/{组件名称}.schema.yaml`

**Schema 结构**：
```yaml
component:
  type: httpsampler
  name: HTTP Request
  description: 发送 HTTP/HTTPS 请求

properties:
  - name: HTTPSampler.protocol
    type: String
    required: false
    enum: ["http", "https", "file"]
    default: "http"

  - name: HTTPSampler.domain
    type: String
    required: false
    description: 域名或 IP 地址

  - name: HTTPSampler.port
    type: Integer
    required: false
    min: 1
    max: 65535
    default: 80

  - name: Assertion.test_strings
    type: Array
    required: false
    description: 多个测试字符串
```

### 2.3 支持的参数类型

| 类型 | Java 检查 | 示例值 |
|------|-----------|--------|
| `String` | `instanceof String` | `"hello"` |
| `Integer` | `instanceof Integer/Long` | `100` |
| `Boolean` | `instanceof Boolean` | `true` |
| `Number` | `instanceof Number` | `3.14` |
| `Object` | `instanceof Map` | `{"key": "value"}` |
| `Array` | `instanceof Iterable or array` | `["a", "b"]` |

### 2.4 支持的校验规则

| 规则 | 类型 | 说明 |
|------|------|------|
| `required` | 全部 | 是否必传参数 |
| `enum` | String | 允许的枚举值列表 |
| `min` | Integer | 最小值 |
| `max` | Integer | 最大值 |
| `pattern` | String | 正则表达式校验 |
| `default` | 全部 | 默认值 |

## 3. 核心类设计

### 3.1 ComponentSchema.java

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/validation/ComponentSchema.java`

```java
public class ComponentSchema {
    private String componentType;
    private String componentName;
    private String description;
    private final List<PropertyDefinition> properties = new ArrayList<>();

    public static class PropertyDefinition {
        private String name;
        private PropertyType type;        // STRING, INTEGER, BOOLEAN, NUMBER, OBJECT, ARRAY
        private boolean required;
        private Object defaultValue;
        private List<String> enumValues;
        private Integer minValue;
        private Integer maxValue;
        private String pattern;
        private String description;
    }

    public enum PropertyType {
        STRING, INTEGER, BOOLEAN, NUMBER, OBJECT, ARRAY
    }
}
```

### 3.2 ComponentSchemaLoader.java

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/validation/ComponentSchemaLoader.java`

**职责**：
- 从 `references/` 目录加载 `.schema.yaml` 文件
- 使用 **SnakeYAML** 解析 YAML
- 缓存已加载的 schema（按标准化组件类型索引）

**关键方法**：
- `loadSchema(String elementType)`: 返回 ComponentSchema 或 null（未配置时）
- `loadAllSchemas()`: 启动时扫描所有 schema 文件

### 3.3 ComponentValidator.java

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/validation/ComponentValidator.java`

**校验顺序**：
1. **未知参数检查** - 参数是否在 schema 中定义（显示可用参数列表）
2. **必传参数检查** - required 参数是否存在
3. **类型检查** - 参数值类型是否正确
4. **值检查** - 枚举、范围、正则等约束

**向后兼容**：如果组件没有 schema，返回 `ValidationResult.valid()`

### 3.4 CreateJMeterElementTool.java 集成

**路径**：`src/main/java/org/qainsights/jmeter/ai/agent/tools/jmeter/CreateJMeterElementTool.java`

**集成点**：在 `executeInternal()` 方法中，基本校验之后、组件创建之前

```java
// Schema validation (if schema exists)
if (componentValidator != null) {
    ValidationResult validation = componentValidator.validate(elementType, properties);
    if (!validation.isValid()) {
        String errorMsg = buildValidationErrorMessage(elementType, validation);
        return ToolResult.error(errorMsg);
    }
}
```

## 4. 数组属性支持

### 4.1 背景

用户需要传递数组类型的属性，如 `Assertion.test_strings=["200", "301"]`，这在 JMeter 中对应 `CollectionProperty`。

### 4.2 实现方式

在 `CreateJMeterElementTool.setElementProperties()` 中添加数组检测和处理逻辑：

```java
// 使用反射创建 CollectionProperty 和 StringProperty
Class<?> collectionPropClass = Class.forName("org.apache.jmeter.testelement.property.CollectionProperty");
Class<?> stringPropClass = Class.forName("org.apache.jmeter.testelement.property.StringProperty");

// 创建 CollectionProperty（需要两个参数）
Object collectionProp = collectionPropClass
    .getConstructor(String.class, java.util.Collection.class)
    .newInstance(propName, new java.util.ArrayList<>());

// 使用 addProperty 方法添加元素
java.lang.reflect.Method addPropertyMethod = collectionPropClass.getMethod("addProperty", jmeterPropClass);
for (String item : items) {
    Object stringProp = stringPropClass
        .getConstructor(String.class, String.class)
        .newInstance(uniqueName, item);
    addPropertyMethod.invoke(collectionProp, stringProp);
}
```

### 4.3 生成的 XML 结构

```xml
<collectionProp name="Assertion.test_strings">
  <stringProp name="1712478901123">200</stringProp>
  <stringProp name="1712478901124">301</stringProp>
</collectionProp>
```

| 嵌套结构 | 不支持 | 完全支持 |
| 依赖 | 无 | 1 个轻量库 |

## 7. 遇到的问题和解决方案

### 7.1 PropertyDefinition.getName() 返回 null

**问题**：YAML 解析时某些属性定义的 name 字段为空

**解决**：
1. 在 `parsePropertyField` 中跳过空 name
2. 在 `getProperty` 中添加 null 检查
3. 在 `getRequiredProperties` 中过滤无效属性

### 7.2 properties 空值处理

**问题**：`Map.of()` 创建的不可变 Map 导致 NPE

**解决**：改用 `new HashMap<>()` 创建可变 Map

### 7.3 CollectionProperty 构造函数错误

**问题**：`NoSuchMethodException: CollectionProperty.<init>(java.lang.String)`

**解决**：使用两参数构造函数 `CollectionProperty(String name, Collection<?> value)`

### 7.4 StringProp 类名错误

**问题**：`ClassNotFoundException: StringProp`

**解决**：正确类名是 `StringProperty`

### 7.5 addItem 方法签名错误

**问题**：`NoSuchMethodException: addItem(JMeterProperty)`

**解决**：使用 `addProperty(JMeterProperty)` 方法而不是 `addItem`

## 8. 校验错误消息示例

### 8.1 缺少必传参数

```
**Component Parameter Validation Failed**

Component Type: responseassertion

Errors:
  - Missing required parameter 'Assertion.test_field'. What to test (response code, data, headers, etc.)

**Solution**: Please provide all required parameters with correct values.
```

### 8.2 未知参数（显示可用参数）

```
**Component Parameter Validation Failed**

Component Type: httpsampler

Errors:
  - Unknown parameter(s) [WrongParam, InvalidParam] are not defined in the component schema. Available parameters: [HTTPSampler.connect_timeout, HTTPSampler.domain, HTTPSampler.method, ...]

**Solution**: Please provide all required parameters with correct values.
```

### 8.3 参数值类型错误

```
**Component Parameter Validation Failed**

Component Type: httpsampler

Errors:
  - Parameter 'HTTPSampler.port' has invalid type. Expected INTEGER but got String

**Solution**: Please provide all required parameters with correct values.
```

### 8.4 枚举值不在列表内

```
**Component Parameter Validation Failed**

Component Type: responseassertion

Errors:
  - Parameter 'Assertion.test_field' has invalid value 'Invalid'. Allowed values: [Assertion.response_data, Assertion.response_code, Assertion.response_message, ...]

**Solution**: Please provide all required parameters with correct values.
```

## 9. 使用示例

### 9.1 创建响应断言

```json
{
  "elementType": "responseassertion",
  "elementName": "断言_状态码200或301",
  "properties": {
    "Assertion.test_field": "Assertion.response_code",
    "Assertion.test_type": 8,
    "Assertion.test_strings": ["200", "301"]
  }
}
```

### 9.2 创建 HTTP 请求

```json
{
  "elementType": "httpsampler",
  "elementName": "GET_查询用户",
  "properties": {
    "HTTPSampler.domain": "api.example.com",
    "HTTPSampler.protocol": "https",
    "HTTPSampler.path": "/api/users",
    "HTTPSampler.method": "GET",
    "HTTPSampler.port": 443
  }
}
```

## 10. 后续优化方向

### 10.1 功能扩展

1. **条件必传参数**：如 "domain 或 path 至少一个必传"
2. **参数依赖关系**：如 "postBodyRaw=true 时，Arguments 只能有一个元素"
3. **正则表达式校验**：已支持 `pattern` 字段
4. **自定义校验器**：允许为特定参数定义自定义校验逻辑

### 10.2 Schema 管理

1. **Schema 生成工具**：从 JMeter 源码自动生成 schema
2. **Schema 验证工具**：检查 schema 文件的正确性
3. **Schema 文档生成**：自动生成可读的参数文档

### 10.3 性能优化

1. **Schema 缓存**：已实现，使用 Map 缓存
2. **延迟加载**：按需加载 schema 而非启动时全部加载
3. **增量更新**：支持热更新 schema 文件
